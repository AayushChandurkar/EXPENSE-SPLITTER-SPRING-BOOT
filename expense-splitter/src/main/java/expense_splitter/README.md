# Expense Splitter - REST API Backend

A production-ready RESTful backend API for an expense splitting application inspired by Splitwise. Built with Java, Spring Boot, Spring Security, and MySQL.

## Features

- JWT based stateless authentication
- User registration and login with BCrypt password hashing
- Group management with member addition and removal
- Expense tracking with automatic equal split calculation
- Settlement tracking per member per expense
- Global exception handling with clean error responses
- Input validation using Bean Validation

## Architecture

The project follows a clean layered architecture:
Controller → Service → Repository → Database

- **Controller** — Handles HTTP requests and responses
- **Service** — Contains all business logic
- **Repository** — Database operations via Spring Data JPA
- **Entity** — Database table mappings
- **DTO** — Request and response objects
- **Security** — JWT filter, configuration and utilities
- **Exception** — Global exception handling

## Database Schema

users
├── id, name, email, password, created_at
expense_groups
├── id, name, description, created_by(FK), created_at
group_members (join table)
├── user_id(FK), group_id(FK)
expenses
├── id, description, amount, group_id(FK), paid_by(FK), created_at
expense_members
├── id, expense_id(FK), user_id(FK), share_amount, settled

## API Endpoints

### Auth
| Method | Endpoint | Description |
|---|---|---|
| POST | /api/auth/register | Register new user |
| POST | /api/auth/login | Login and get JWT token |

### Groups
| Method | Endpoint | Description |
|---|---|---|
| POST | /api/groups | Create a group |
| GET | /api/groups | Get all my groups |
| GET | /api/groups/{id} | Get group by ID |
| POST | /api/groups/{id}/members | Add member to group |
| DELETE | /api/groups/{id}/members/{userId} | Remove member |

### Expenses
| Method | Endpoint | Description |
|---|---|---|
| POST | /api/groups/{id}/expenses | Add expense to group |
| GET | /api/groups/{id}/expenses | Get group expenses |
| GET | /api/expenses/{id} | Get expense by ID |
| DELETE | /api/expenses/{id} | Delete expense |
| POST | /api/expenses/{id}/settle | Settle your share |

## Getting Started

### Prerequisites
- Java 17+
- MySQL 8+
- Maven

### Setup

1. Clone the repository
```bash
git clone https://github.com/AayushChandurkar/EXPENSE-SPLITTER-SPRING-BOOT.git
```

2. Create MySQL database
```sql
CREATE DATABASE expense_splitter;
```

3. Configure application properties
```bash
cp src/main/resources/application.properties.example src/main/resources/application.properties
```

Edit `application.properties` with your MySQL credentials.

4. Run the application
```bash
mvn spring-boot:run
```

App runs on `http://localhost:8080`

## How It Works

### Authentication Flow

Register → BCrypt hashes password → Save to DB
Login → Verify credentials → Generate JWT token
Every request → JWT Filter validates token → Allow or reject

### Split Calculation

Expense amount / number of members = share per person
Example: 1200 / 3 = 400.00 per person
Each person gets an ExpenseMember record with their share

### Balance Logic

Net position = Amount Paid - Share Amount
Positive = someone owes you money
Negative = you owe someone money
